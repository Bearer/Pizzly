In this 5min Quickstart, you will: 
1. authorize an example API 
2. continuously sync data from that API

## Authorize an API (Github)

Create a free Nango account (no credit card required):
[![Try Nango Cloud](/images/nango-deploy-button.svg)](https://app.nango.dev/signup)

On the [Integrations tab](https://app.nango.dev), with the left bar environment toggle set to `Dev`, click `OAuth` next to the example Github integration:

<Frame>
  <img src="/images/quickstart/nango-oauth-button.png" />
</Frame>

On the `Add New Connection` page, keep the default values and click `Start OAuth Flow`.

Authorize Github and retrieve your credentials from the [Connections tab](https://app.nango.dev/connections) ðŸŽ‰

<Tip>
If you are using Nango to authorize APIs, that's it ðŸ™Œ Check the [API Authorization guide](/guides/oauth) to authorize users from your frontend and get credentials from your backend.
</Tip>

</Info>

## Sync data from an API (Github issues)

### Install the Nango CLI

Install the Nango CLI globally: 
```bash
npm install nango -g
```

In the folder where you want your integration folder (e.g. root of your project), run:
```bash
nango init # Creates `/nango-integrations` with initial config
```

Next, we need to authenticate the CLI. Add the following env vars (e.g. in an `.env` file in `/nango-integrations`):
```bash
NANGO_SECRET_KEY_PROD='<prod-secret-key>'
NANGO_SECRET_KEY_DEV='<dev-secret-key>'
```

Get your `prod` and `dev` secret keys from the [Project Settings tab](https://app.nango.dev/project-settings) (toggle between the `prod` and `dev` environment in the left nav bar).

### Configure a Sync

`/nango-integrations` initially contains two files:
- `nango.yaml`: the **configuration** to sync Github issues
- `github-issue-example.ts`: the **code** to sync Github issues

Within `/nango-integrations`, run:
```bash
nango dev # Continuously watches your integration files for changes.
```

The rest of the files (`models.ts`, `dist/*`) are automatically generated thanks to this `dev` command.

### Dry run the Sync

Open a new terminal tab, go to `/nango-integrations`. Dry run your sync by running:

```bash
nango dryrun github-issue-example test-connection-id
```

This will use the Connection you created in the [Authorize an API section](#authorize-an-api-github) to run your Sync script. Synced objects will not be persisted in Nango's storage. It allows you to quickly check if your Sync script is correct.

### Deploy the Sync

Finally, deploy your new configuration:

```bash
nango deploy dev # Deploys to your cloud "dev" environment.
```

Your Sync will run on Nango's cloud infrastructure, syncing Github issues for all the Github Connections, continuously ðŸŽ‰ You can manage your Syncs from the [Syncs tab](https://app.nango.dev/syncs).

### Retrieve the synced data

Fetch the always-fresh data from the Sync in your backend with the [backend SDK](/sdks) or the [REST API](/api-reference):

<Tabs>

<Tab title="Node SDK">

```ts
import { Nango }  from '@nangohq/node';

const nango = new Nango({ secretKey: '<your-dev-secret-key>' });

const result = await nango.getRecords({
    providerConfigKey: 'github',
    connectionId: 'test-connection-id',
    model: 'GithubIssue'
});
```
</Tab>

<Tab title="cURL">

```bash
curl --request GET \
  --url https://api.nango.dev/sync/records?model=GithubIssue \
  --header 'Authorization: Bearer <your-dev-secret-key>' \
  --header 'Connection-Id: test-connection-id' \
  --header 'Provider-Config-Key: github'
```
</Tab>

</Tabs>
