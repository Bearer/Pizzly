# Overview

To authenticate with both Builder.io (public) and Builder.io (private), you will need one key pieces of information:
1. **API Key** - A key that grants Nango permission to interact with  Builder.io's APIs resources and services.

This guide will walk you through finding the API Keys within Builder.io.

### Prerequisites:

- You must have a registered account with  Builder.io.

### Instructions:

#### Step 1: Finding Your API Domain
1. Different APIs within Builder.io require different domains, i.e `Content API` uses the `cdn.builder.io` domain, for more on this please visit [API Intro](https://www.builder.io/c/docs/api-intro)


#### Step 2: Finding Your API Key
1. **Log in** to your Builder.io dashboard.
2. Select your space then go to the **Space settings** section, under the **Space** tab, you can view both your **public** and **private** api keys.

<img src="/integrations/all/builder-io/builder_io_api_key.png" />


#### Step 3: Enter credentials in the Connect UI

Once you have the **API Key**:
1. Open the form where you need to authenticate with Builder.io.
2. Enter both the **API Key** and **Domain** in the designated fields.
3. Submit the form, and you should be successfully authenticated.

<img src="/integrations/all/builder-io/form.png" style={{maxWidth: "450px" }}/>

You are now connected to Builder.io.
