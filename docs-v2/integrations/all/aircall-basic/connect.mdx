# Overview

To authenticate with Aircall, you need two key pieces of information:
1. **Application ID** - A unique identifier for your application on Aircall.
2. *API Token** - A key that grants full access to all your application and configurations.

This guide will walk you through finding or creating those credentials within Aircall.


### Prerequisites:

- You must set up an account with Aircall.
- You should have an associated Aircall number.

#### Step 1: Finding Your Admin API Key (API Token) and Application ID (API ID)
1. Log into your admin aircall account and on the dashboard navigate to **Integrations & API** option.

<img src="/integrations/all/aircall/nango_aircall_integration_tab.png" />

2. To generate an API, navigate to the **API Key** tab

<img src="/integrations/all/aircall/nango_aircall_gen_key.png" />

3. After creating the API Token, a modal with the **API ID** and **API Token** will be in view.

NOTE: Securely copy the API Token, it will not be available for view after this modal is closed.

<img src="/integrations/all/aircall/nango_aircall_Token_API_ID.png" />

#### Step 2: Enter credentials in the Connect UI

Once you have the **API Token** and **API ID**:

<img src="/integrations/all/aircall/nango_aircall - basic.png" />

1. Open the form where you need to authenticate with Aircall - basic.
1. Enter the **API Token** and **API ID**.
1. **IMPORTANT**: the `User Name` field coresponds to the **API ID** and `Password` is **API Token**
1. Submit the form, and you should be successfully authenticated.

You are now connected to Aircall - basic.
